## Installation

```package-install
npm install @hot-updater/cloudflare --save-dev
```

## Setup

The easiest way to set up your backend is using the init command:

```package-install
npx hot-updater init
```

This interactive command will guide you through the setup process. For details on what the init command does, see the [Cloudflare](/docs/managed/cloudflare) documentation.

For manual configuration, use the settings below.

## Configuration

```typescript
interface D1DatabaseConfig {
  dialect: Dialect; // Kysely dialect for the D1 binding
}
```

## Usage

In a Cloudflare Worker, use the Worker helper. It supplies the Kysely D1
dialect from the `env.DB` binding at request time.

```typescript
import { createHotUpdater } from "@hot-updater/server";
import { d1Database } from "@hot-updater/cloudflare/worker";

export const hotUpdater = createHotUpdater({
  database: d1Database(),
  // ... storage and routes
});
```

If you are composing the database manually, pass a Kysely D1 dialect yourself.

```typescript
import { d1Database } from "@hot-updater/cloudflare";
import { D1Dialect } from "kysely-d1";

export const database = d1Database({
  dialect: new D1Dialect({ database: env.DB }),
});
```

## Complete Example

Combined R2 storage and D1 database in a Worker:

```typescript
import { createHotUpdater } from "@hot-updater/server";
import { d1Database, r2Storage } from "@hot-updater/cloudflare/worker";

export const hotUpdater = createHotUpdater({
  database: d1Database(),
  storages: [
    r2Storage({
      publicBaseUrl: ({ request }) => new URL(request.url).origin,
    }),
  ],
  basePath: "/api/check-update",
  routes: {
    updateCheck: true,
    bundles: false,
  }),
});
```
