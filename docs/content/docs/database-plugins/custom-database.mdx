## Installation

```package-install
npm install @hot-updater/plugin-core --save-dev
```

## Overview

Build a custom database plugin to integrate any database provider. This guide shows how to use `createDatabasePlugin` to build and configure your own database plugin.

## Creating a Database Plugin

Use `createDatabasePlugin` to build custom database plugins:

```typescript
import { createDatabasePlugin } from "@hot-updater/plugin-core";

export const myDatabase = createDatabasePlugin({
  name: "myDatabase",              // Plugin identifier
  connect: (config: MyConfig) => ({
    bundles: {
      getById: async ({ bundleId }) => { /* ... */ },
      findMany: async ({ where, orderBy, window }) => [],
      count: async ({ where }) => 0,
      insert: async ({ bundle }) => { /* ... */ },
      update: async ({ bundleId, patch }) => { /* ... */ },
      delete: async ({ bundleId }) => { /* ... */ },
    },
    bundlePatches: {
      getById: async ({ patchId }) => { /* ... */ },
      findMany: async ({ where, orderBy, window }) => [],
      count: async ({ where }) => 0,
      insert: async ({ patch }) => { /* ... */ },
      update: async ({ patchId, patch }) => { /* ... */ },
      delete: async ({ patchId }) => { /* ... */ },
    },
    close: async () => { /* optional cleanup */ },
  })
});
```

### Connect Function Return Type

Your `connect` function must return a resource core with these methods:

```typescript
{
  bundles: {
    getById: (params: { bundleId: string }) =>
      Promise<DatabaseBundleRecord | null>;
    findMany: (query: BundleFindManyQuery) =>
      Promise<readonly DatabaseBundleRecord[]>;
    count: (query: BundleCountQuery) => Promise<number>;
    insert: (params: { bundle: DatabaseBundleRecord }) => Promise<void>;
    update: (params: {
      bundleId: string;
      patch: Partial<DatabaseBundleRecord>;
    }) => Promise<void>;
    delete: (params: { bundleId: string }) => Promise<void>;
  };

  bundlePatches: {
    getById: (params: { patchId: string }) =>
      Promise<DatabaseBundlePatch | null>;
    findMany: (query: BundlePatchFindManyQuery) =>
      Promise<readonly DatabaseBundlePatch[]>;
    count: (query: BundlePatchCountQuery) => Promise<number>;
    insert: (params: { patch: DatabaseBundlePatch }) => Promise<void>;
    update: (params: {
      patchId: string;
      patch: DatabaseBundlePatchUpdate;
    }) => Promise<void>;
    delete: (params: { patchId: string }) => Promise<void>;
  };

  updateInfo?: {
    get: (args: GetBundlesArgs) => Promise<UpdateInfo | null>;
  };

  bundleEvents?: {
    findMany: (query: BundleEventFindManyQuery) =>
      Promise<readonly DatabaseBundleEvent[]>;
    count: (query: BundleEventCountQuery) => Promise<number>;
    append: (params: { event: DatabaseBundleEvent }) => Promise<void>;
  };

  beginTransaction?: () => Promise<DatabaseTransaction>;
  close?: () => Promise<void>;
}
```

Implement `updateInfo.get` whenever your backend can answer update checks
without scanning the full management list. This is especially important for
blob/object-storage plugins, where the hot path should read only the manifest
objects needed for the request.

`createDatabasePlugin` wraps the core with a runtime identity map and generates
the public `commit()` method. Provider resources persist concrete resource
writes only; they do not implement Hot Updater workflows such as deploy,
rollback, promote, or bundle delete.

## Implementation Example

Here's a complete custom database plugin using a REST API:

```typescript title="customDatabase.ts"
import {
  type BundleCountQuery,
  type BundleFindManyQuery,
  type BundlePatchCountQuery,
  type BundlePatchFindManyQuery,
  type DatabaseBundlePatch,
  type DatabaseBundlePatchUpdate,
  type DatabaseBundleRecord,
  createDatabasePlugin,
} from "@hot-updater/plugin-core";

export interface CustomDatabaseConfig {
  baseUrl: string;
  apiKey: string;
}

export const customDatabase = createDatabasePlugin({
  name: "customDatabase",
  connect: (config: CustomDatabaseConfig) => {
    const headers = {
      "Content-Type": "application/json",
      Authorization: `Bearer ${config.apiKey}`,
    };

    return {
      bundles: {
        async getById({ bundleId }) {
          const response = await fetch(`${config.baseUrl}/bundles/${bundleId}`, {
            headers,
          });

          if (!response.ok) {
            if (response.status === 404) return null;
            throw new Error(`Failed to fetch bundle: ${response.statusText}`);
          }

          return response.json();
        },

        async findMany(
          query: BundleFindManyQuery,
        ): Promise<readonly DatabaseBundleRecord[]> {
          const params = new URLSearchParams({
            limit: String(query.window.limit),
            offset: String(query.window.offset),
          });

          if (query.where?.channel) {
            params.set("channel", query.where.channel);
          }
          if (query.where?.platform) {
            params.set("platform", query.where.platform);
          }

          const response = await fetch(`${config.baseUrl}/bundles?${params}`, {
            headers,
          });

          if (!response.ok) {
            throw new Error(`Failed to fetch bundles: ${response.statusText}`);
          }

          return response.json();
        },

        async count(query: BundleCountQuery): Promise<number> {
          const params = new URLSearchParams();
          if (query.where?.channel) {
            params.set("channel", query.where.channel);
          }
          if (query.where?.platform) {
            params.set("platform", query.where.platform);
          }

          const response = await fetch(`${config.baseUrl}/bundles/count?${params}`, {
            headers,
          });

          if (!response.ok) {
            throw new Error(`Failed to count bundles: ${response.statusText}`);
          }

          const { count } = await response.json();
          return count;
        },

        async insert({ bundle }) {
          await fetch(`${config.baseUrl}/bundles`, {
            method: "POST",
            headers,
            body: JSON.stringify(bundle),
          });
        },

        async update({ bundleId, patch }) {
          await fetch(`${config.baseUrl}/bundles/${bundleId}`, {
            method: "PATCH",
            headers,
            body: JSON.stringify(patch),
          });
        },

        async delete({ bundleId }) {
          await fetch(`${config.baseUrl}/bundles/${bundleId}`, {
            method: "DELETE",
            headers,
          });
        },
      },

      bundlePatches: {
        async getById({ patchId }) {
          const response = await fetch(
            `${config.baseUrl}/bundle-patches/${patchId}`,
            { headers },
          );

          if (!response.ok) {
            if (response.status === 404) return null;
            throw new Error(`Failed to fetch patch: ${response.statusText}`);
          }

          return response.json();
        },

        async findMany(
          query: BundlePatchFindManyQuery,
        ): Promise<readonly DatabaseBundlePatch[]> {
          const params = new URLSearchParams({
            limit: String(query.window.limit),
            offset: String(query.window.offset),
          });
          if (query.where?.bundleId) {
            params.set("bundleId", query.where.bundleId);
          }

          const response = await fetch(
            `${config.baseUrl}/bundle-patches?${params}`,
            { headers },
          );

          if (!response.ok) {
            throw new Error(`Failed to fetch patches: ${response.statusText}`);
          }

          return response.json();
        },

        async count(query: BundlePatchCountQuery): Promise<number> {
          const params = new URLSearchParams();
          if (query.where?.bundleId) {
            params.set("bundleId", query.where.bundleId);
          }

          const response = await fetch(
            `${config.baseUrl}/bundle-patches/count?${params}`,
            { headers },
          );

          if (!response.ok) {
            throw new Error(`Failed to count patches: ${response.statusText}`);
          }

          const { count } = await response.json();
          return count;
        },

        async insert({ patch }) {
          await fetch(`${config.baseUrl}/bundle-patches`, {
            method: "POST",
            headers,
            body: JSON.stringify(patch),
          });
        },

        async update({
          patchId,
          patch,
        }: {
          patchId: string;
          patch: DatabaseBundlePatchUpdate;
        }) {
          await fetch(`${config.baseUrl}/bundle-patches/${patchId}`, {
            method: "PATCH",
            headers,
            body: JSON.stringify(patch),
          });
        },

        async delete({ patchId }) {
          await fetch(`${config.baseUrl}/bundle-patches/${patchId}`, {
            method: "DELETE",
            headers,
          });
        },
      },
    };
  },
});
```

Use `window.offset` and `window.limit` as the provider read contract. The public
runtime still exposes cursor pagination, and `createDatabasePlugin` converts
runtime cursors into provider windows.

Callers stage writes through resource methods and then flush them with the
runtime `commit()` method:

```typescript
const database = await customDatabase({
  baseUrl: process.env.DATABASE_BASE_URL!,
  apiKey: process.env.DATABASE_API_KEY!,
});

await database.bundles.insert({ bundle });
for (const patch of patches) {
  await database.bundlePatches.insert({ patch });
}
await database.commit();
```

## Helper Utilities

The `@hot-updater/plugin-core` package owns runtime pagination for database
plugins. Provider implementations normally only need to return rows from
`findMany` and totals from `count`.

### calculatePagination

Generates pagination metadata for runtime/public pagination helpers:

```typescript
import { calculatePagination } from "@hot-updater/plugin-core";

const startIndex = resolvedCursorStartIndex;
const pagination = calculatePagination(totalCount, {
  limit: 10,
  offset: startIndex,
});
// Result: { total, hasNextPage, hasPreviousPage, currentPage, totalPages }
```

```typescript
import { defineConfig } from "@hot-updater/core";
import { customDatabase } from "./customDatabase";

export default defineConfig({
  database: customDatabase({
    baseUrl: process.env.DATABASE_BASE_URL!,
    apiKey: process.env.DATABASE_API_KEY!,
  }),
  // ... other config
});
```

## Best Practices

### Security

- Never hardcode credentials in plugin code
- Use environment variables for sensitive data
- Implement proper authentication headers
- Validate input data before storing

### Performance

- Prefer `getUpdateInfo` for the update-check hot path so runtime requests do
  not fall back to wide bundle scans
- Implement efficient filtering in `findMany` and `count`
- Use database indexes for common queries
- Use `beginTransaction` or `commit({ batch })` support when your provider can
  apply staged mutations atomically
- Cache channel lists if they don't change frequently
- Add deterministic I/O or query-counter tests that fail if update checks or
  cursor-backed management reads regress into broad scans

### Error Handling

- Return `null` from `getBundleById` when not found (don't throw)
- Throw descriptive errors for other failures
- Handle network errors gracefully
- Validate bundle data structure
