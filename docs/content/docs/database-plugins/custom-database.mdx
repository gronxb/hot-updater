## Installation

```package-install
npm install @hot-updater/server kysely --save-dev
```

## Overview

Custom database integrations should start from one of the official database
middle layers:

- Kysely for provider packages and dialect-backed SQL databases.
- Drizzle for applications that already own a Drizzle client.
- Prisma for applications that already own a Prisma client.

Those middle layers keep the Hot Updater database contract stable while hiding
the low-level runtime factory from normal provider authoring. A provider should
focus on the database capability it owns, such as a Kysely dialect, connection
factory, migration path, and explicit Hot Updater provider metadata.

## Provider Shape

A custom provider is expected to expose a configured database function from its
package, then pass the result to Hot Updater configuration.

```typescript
import { defineConfig } from "@hot-updater/core";
import { customKyselyDatabase } from "./customKyselyDatabase";

export default defineConfig({
  database: customKyselyDatabase({
    connectionString: process.env.DATABASE_URL,
  }),
});
```

Use the same configured provider with `createHotUpdater` for self-hosted
servers:

```typescript
import { createHotUpdater } from "@hot-updater/server";
import { customKyselyDatabase } from "./customKyselyDatabase";

const hotUpdater = createHotUpdater({
  database: customKyselyDatabase({
    connectionString: process.env.DATABASE_URL,
  }),
});
```

## Database Responsibilities

Custom providers should keep these responsibilities explicit:

- Provider identity and CLI adapter names stay Hot Updater metadata.
- Kysely dialects describe database communication, not Hot Updater provider
  identity.
- Migration and schema generation behavior must be documented by the provider.
- Connections should define clear ownership for pool shutdown and transaction
  support.
- Non-SQL backends should be documented as native or legacy integrations unless
  they are actually implemented through an official middle layer.

## Best Practices

- Prefer a Kysely-backed provider when building a new first-party or community
  SQL integration.
- Prefer Drizzle or Prisma middle layers when your application already manages
  those clients directly.
- Keep credentials in environment variables or platform secrets.
- Add database indexes for fields used by update-check lookups.
- Test CLI generate and migrate behavior alongside runtime database operations.
