## Installation

```package-install
npm install @hot-updater/plugin-core --save-dev
```

## Overview

Build a custom storage plugin to integrate any storage provider.

Most apps do not need this guide. Use it when you are writing or maintaining a
storage plugin.

Storage plugins are split into explicit environment profiles so plugin authors
can implement the APIs that are safe for each runtime.

- `node`: used by the CLI, deploy command, and console. It can read and write local files.
- `runtime`: used by update-check servers and edge runtimes. It must resolve HTTP(S) bundle URLs and read small text artifacts such as `manifest.json` directly from storage.

Use the narrow helper that matches your plugin:

| Helper | Use when |
| --- | --- |
| `createNodeStoragePlugin` | The plugin is only used by deploy-time tooling. |
| `createRuntimeStoragePlugin` | The plugin is only used by an update-check runtime. |
| `createFullStoragePlugin` | The same provider implementation supports both deploy and runtime usage. |
| `createStoragePlugin` | You need a custom profile combination. Prefer the helpers above for public plugins. |

## Creating a Storage Plugin

Use `createFullStoragePlugin` when one plugin can be used by both `hot-updater deploy` and `createHotUpdater`:

```typescript
import { createFullStoragePlugin } from "@hot-updater/plugin-core";

export const myStorage = createFullStoragePlugin<MyConfig>({
  name: "myStorage",
  supportedProtocol: "custom",
  factory: (config) => ({
    node: {
      upload: async (key, filePath) => ({ storageUri: "..." }),
      delete: async (storageUri) => {
        /* ... */
      },
      downloadFile: async (storageUri, filePath) => {
        /* ... */
      },
    },
    runtime: {
      getDownloadUrl: async (storageUri, context) => ({ fileUrl: "..." }),
      readText: async (storageUri, context) => "{...}",
    },
  }),
});
```

### Factory Function Return Type

Your factory function returns one or more profiles.

```typescript
{
  node: {
    // Uploads a local file and returns the storage URI saved in the database.
    upload: (key: string, filePath: string) => Promise<{ storageUri: string }>;
    // Deletes a storage object or storage prefix.
    delete: (storageUri: string) => Promise<void>;
    // Downloads a storage object to a local file path.
    downloadFile: (storageUri: string, filePath: string) => Promise<void>;
  };
  runtime: {
    // Generates an HTTP(S) URL that clients can download.
    getDownloadUrl: (
      storageUri: string,
      context?: HotUpdaterContext
    ) => Promise<{ fileUrl: string }>;
    // Reads a small text artifact directly from the provider.
    readText: (
      storageUri: string,
      context?: HotUpdaterContext
    ) => Promise<string | null>;
  };
}
```

`runtime.readText` is required for runtime storage plugins so update-check
servers can read `manifest.json` and other small metadata directly from storage.
Implement it with the provider SDK, platform binding, or authenticated runtime
API. Do not generate a public download URL and fetch that URL again.

## Implementation Example

Here's a complete custom storage plugin implementation:

```typescript title="customStorage.ts"
import {
  DeleteObjectCommand,
  GetObjectCommand,
  S3Client,
} from "@aws-sdk/client-s3";
import { Upload } from "@aws-sdk/lib-storage";
import { getSignedUrl } from "@aws-sdk/s3-request-presigner";
import {
  createFullStoragePlugin,
  getContentType,
  parseStorageUri,
} from "@hot-updater/plugin-core";
import fs from "fs/promises";
import path from "path";

export interface CustomStorageConfig {
  region: string;
  credentials: {
    accessKeyId: string;
    secretAccessKey: string;
  };
  bucketName: string;
}

export const customStorage = createFullStoragePlugin<CustomStorageConfig>({
  name: "customStorage",
  supportedProtocol: "s3",
  factory: (config) => {
    const { bucketName, ...s3Config } = config;
    const client = new S3Client(s3Config);

    const assertStorageUri = (storageUri: string) => {
      const parsed = parseStorageUri(storageUri, "s3");

      if (parsed.bucket !== bucketName) {
        throw new Error(`Bucket mismatch: expected "${bucketName}"`);
      }

      return parsed;
    };

    return {
      node: {
        async upload(key, filePath) {
          const Body = await fs.readFile(filePath);
          const ContentType = getContentType(filePath);
          const filename = path.basename(filePath);
          const Key = `${key}/${filename}`;

          const upload = new Upload({
            client,
            params: {
              Bucket: bucketName,
              Key,
              Body,
              ContentType,
            },
          });

          await upload.done();

          return {
            storageUri: `s3://${bucketName}/${Key}`,
          };
        },

        async delete(storageUri) {
          const { key } = assertStorageUri(storageUri);

          const command = new DeleteObjectCommand({
            Bucket: bucketName,
            Key: key,
          });

          await client.send(command);
        },

        async downloadFile(storageUri, filePath) {
          const { key } = assertStorageUri(storageUri);
          const response = await client.send(
            new GetObjectCommand({ Bucket: bucketName, Key: key }),
          );
          const bytes = await response.Body?.transformToByteArray();

          if (!bytes) {
            throw new Error("Object body is empty");
          }

          await fs.writeFile(filePath, bytes);
        },
      },
      runtime: {
        async getDownloadUrl(storageUri) {
          const { key } = assertStorageUri(storageUri);

          const command = new GetObjectCommand({
            Bucket: bucketName,
            Key: key,
          });
          const signedUrl = await getSignedUrl(client, command, {
            expiresIn: 3600,
          });

          return { fileUrl: signedUrl };
        },

        async readText(storageUri) {
          const { key } = assertStorageUri(storageUri);
          const response = await client.send(
            new GetObjectCommand({ Bucket: bucketName, Key: key }),
          );

          return response.Body?.transformToString() ?? null;
        },
      },
    };
  },
});
```

## Helper Utilities

The `@hot-updater/plugin-core` package provides helper functions:

### parseStorageUri

Parses storage URIs into bucket and key components:

```typescript
const { bucket, key } = parseStorageUri(
  "s3://my-bucket/path/file.bundle",
  "s3"
);
// bucket: "my-bucket"
// key: "path/file.bundle"
```

### getContentType

Returns MIME type based on file extension:

```typescript
const contentType = getContentType("bundle.js");
// Result: "application/javascript"
```

## CLI Configuration

Use your custom plugin in `hot-updater.config.ts`:

```typescript
import { defineConfig } from "@hot-updater/core";
import { customStorage } from "./customStorage";

export default defineConfig({
  storage: customStorage({
    region: "us-east-1",
    credentials: {
      accessKeyId: process.env.AWS_ACCESS_KEY_ID!,
      secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY!,
    },
    bucketName: process.env.BUCKET_NAME!,
  }),
  // ... other config
});
```

## Custom Server Usage

Use your plugin with `createHotUpdater` for self-hosted servers. The plugin passed to `storages` must implement the `runtime` profile.

```typescript
import { createHotUpdater } from "@hot-updater/server/runtime";
import { customStorage } from "./customStorage";

const hotUpdater = createHotUpdater({
  storages: [
    customStorage({
      region: "us-east-1",
      credentials: {
        accessKeyId: process.env.AWS_ACCESS_KEY_ID!,
        secretAccessKey: process.env.AWS_SECRET_ACCESS_KEY!,
      },
      bucketName: process.env.BUCKET_NAME!,
    }),
  ],
  // ... other options
});
```

### Security

- Never hardcode credentials in plugin code
- Use environment variables for sensitive data
- Validate storage URIs before processing
- Implement proper authentication headers
- Keep `runtime.getDownloadUrl` limited to HTTP(S) client downloads
- Keep `runtime.readText` on direct provider reads for manifest metadata

### Storage URI Format

Use a consistent URI format: `protocol://bucket/path/to/file`

```typescript
return {
  storageUri: `${supportedProtocol}://${bucketName}/${storageKey}`,
};
```
