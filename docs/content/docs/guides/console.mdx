**The console provides the following features:**

- Rollback
- Force update flag
- Quick access to git history for updates
- Bundle filtering, editing, deletion, promotion, and download URLs

## Local Console

For local bundle management, run the CLI console from your React Native project:

```package-install
npx hot-updater console
```

The CLI opens a local server based on the plugins configured in
`hot-updater.config.ts`.

## Deployable Console

For a hosted console, use the deployable Vite/Nitro shell:

```bash
git clone https://github.com/hot-updater/console
cd console
corepack enable
pnpm install
```

Configure `hot-updater.config.ts` in the console repository with the same
storage and database plugins used by your OTA deployment:

```ts title="hot-updater.config.ts"
import { s3Database, s3Storage } from "@hot-updater/aws";
import { defineConfig } from "@hot-updater/console/config";

export default defineConfig({
  storage: s3Storage({ bucketName: process.env.HOT_UPDATER_BUCKET! }),
  database: s3Database({ bucketName: process.env.HOT_UPDATER_BUCKET! }),
});
```

The deployable app registers that config through `@hot-updater/console/vite`.
The package owns the console component and API adapter; the deployable app owns
provider selection, credentials, auth, and Nitro deployment.

Register the Vite plugin so `hot-updater.config.ts` becomes an explicit server
bundle input:

```ts title="vite.config.ts"
import { hotUpdaterConsole } from "@hot-updater/console/vite";
import { defineConfig } from "vite";

export default defineConfig({
  plugins: [
    hotUpdaterConsole(),
    // nitro(), tanstackStart(), viteReact(), ...
  ],
});
```

Then load the console API from the virtual server module:

```ts
import { createConsoleApi } from "virtual:hot-updater-console/server-api";

const api = await createConsoleApi();
```

Run locally:

```bash
pnpm dev
```

Build the Nitro output:

```bash
pnpm build
pnpm start
```

### Cloudflare Pages

Nitro supports Cloudflare Pages with the `cloudflare_pages` preset. Configure
Cloudflare Pages with the Nitro Cloudflare preset and Cloudflare Pages build
settings:

```text
Build command:          corepack enable && pnpm install --frozen-lockfile && NITRO_PRESET=cloudflare_pages pnpm build
Build output directory: dist
Root directory:         /
```

References: [Nitro Cloudflare provider](https://nitro.build/deploy/providers/cloudflare)
and [Cloudflare Pages build configuration](https://developers.cloudflare.com/pages/configuration/build-configuration/).

For direct upload:

```bash
NITRO_PRESET=cloudflare_pages pnpm build
pnpm dlx wrangler pages deploy dist
```

Add the environment variables and Cloudflare bindings required by
`hot-updater.config.ts` to the Pages project. For R2-backed storage, bind the
bucket used by the storage plugin. If your deployable console uses TanStack
Start or Better Auth, set the Cloudflare compatibility flag to
`nodejs_compat`.

## Configuration

This is a console configuration that does not affect the bundle.

You can configure additional console settings in the `hot-updater.config.ts` file.

The `storage` and `database` plugins must exist.

| Option           | Description                                              | Default |
| ---------------- | -------------------------------------------------------- | ------- |
| `console.gitUrl` | Enables commit shortcuts by linking to git commit hashes | -       |
| `console.port`   | Sets the console server port                             | `1422`  |

```ts title="hot-updater.config.ts"
import { defineConfig } from "hot-updater";

export default defineConfig({
  console: { // [!code hl]
    port: 3000, // [!code hl]
  }, // [!code hl]
  build: ...,
  storage: ...,
  database: ...,
});

```

You can also use the bundle CLI for automation and CI scripts:

```package-install
npx hot-updater bundle list -p ios
npx hot-updater bundle show <bundle-id>
npx hot-updater bundle disable <bundle-id> -y
npx hot-updater bundle enable <bundle-id> -y
npx hot-updater bundle update <bundle-id> --force-update true -y
npx hot-updater bundle update <bundle-id> --rollout-cohort-count 500 -y
npx hot-updater bundle update <bundle-id> --target-cohorts qa,dogfood -y
```

Add `--json` to `bundle list`, `bundle show`, or `bundle update` when another
tool needs to read the raw bundle data.

## Rollback

When you set a bundle's `enabled` status to `false` in the console, users who have that bundle will immediately force update to the **previous bundle**.

<img src={"/docs/console/enabled.png"} />

## Force Update

When you set a bundle's `shouldForceUpdate` status to `true` in the console, users who have that bundle will immediately force update to the **latest bundle**.

<img src={"/docs/console/force_update.png"} />

## Git URL

You can set the git URL in the `hot-updater.config.ts` file.

```ts title="hot-updater.config.ts"
import { defineConfig } from "hot-updater";

export default defineConfig({
  console: {
    gitUrl: "<your-repo>", // e.g. https://github.com/gronxb/hot-updater // [!code hl]
  },
});
```

When you set the git URL, the commit hash will be clickable and redirect you to the corresponding commit in your repository.
