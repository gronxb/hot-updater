**The console provides the following features:**

- Rollback
- Force update flag
- Quick access to git history for updates
- OTA analytics and installation history on supported databases

## Configuration (Optional)

This is a console configuration that does not affect the bundle.

You can configure additional console settings in the `hot-updater.config.ts` file.

The storage plugin and database adapter must exist.

| Option           | Description                                              | Default |
| ---------------- | -------------------------------------------------------- | ------- |
| `console.gitUrl` | Enables commit shortcuts by linking to git commit hashes | -       |
| `console.port`   | Sets the console server port                             | `1422`  |

```ts title="hot-updater.config.ts"
import { defineConfig } from "hot-updater";

export default defineConfig({
  console: { // [!code hl]
    port: 3000, // [!code hl]
  }, // [!code hl]
  build: ...,
  storage: ...,
  database: ...,
});

```

## Usage

Execute the following command:

```package-install
npx hot-updater console
```

You can also use the bundle CLI for automation and CI scripts:

```package-install
npx hot-updater bundle list -p ios
npx hot-updater bundle show <bundle-id>
npx hot-updater bundle disable <bundle-id> -y
npx hot-updater bundle enable <bundle-id> -y
npx hot-updater bundle update <bundle-id> --force-update true -y
npx hot-updater bundle update <bundle-id> --rollout-cohort-count 500 -y
npx hot-updater bundle update <bundle-id> --target-cohorts qa,dogfood -y
```

Add `--json` to `bundle list`, `bundle show`, or `bundle update` when another
tool needs to read the raw bundle data.

## Rollback

When you set a bundle's `enabled` status to `false` in the console, users who have that bundle will immediately force update to the **previous bundle**.

<img src={"/docs/console/enabled.png"} />

## Force Update

When you set a bundle's `shouldForceUpdate` status to `true` in the console, users who have that bundle will immediately force update to the **latest bundle**.

## Git URL

You can set the git URL in the `hot-updater.config.ts` file.

```ts title="hot-updater.config.ts"
import { defineConfig } from "hot-updater";

export default defineConfig({
  console: {
    gitUrl: "<your-repo>", // e.g. https://github.com/gronxb/hot-updater // [!code hl]
  },
});
```

When you set the git URL, the commit hash will be clickable and redirect you to the corresponding commit in your repository.

## Analytics

Databases that expose the optional Analytics capability add Analytics and
Installations navigation. The Analytics page shows:

- active installations for a 24-hour, 7-day, or 30-day server-receipt window;
- the latest reported bundle for each active installation and its distribution;
- a non-cumulative app-ready activity series;
- all-time Installed and Recovered outcomes for the leading bundle;
- rollout configuration, kept separate from observed reports; and
- an exact, case-sensitive User ID alias filter.

<img
  alt="Analytics overview with activity, latest bundles, outcomes, and rollout configuration"
  src={"/docs/console/analytics-overview.png"}
/>

<img
  alt="Analytics overview on a mobile viewport"
  src={"/docs/console/analytics-overview-mobile.png"}
/>

“Active installation” means a distinct installation ID that sent an app-ready
report during the selected range. It does not mean a person, a device that is
online now, complete fleet coverage, or rollout completion. A user ID is only
searchable/filterable metadata; multiple installations under one user ID still
count separately. Installed and Recovered are transition outcomes, so routine
unchanged app-ready reports do not inflate them.

The shared CRUD aggregation path is bounded to 50,000 matching event rows. A
deployment that exceeds that window needs provider-native aggregation or
dedicated Analytics storage instead of treating the Console as unbounded.

Use the Installations page to search by username, user ID, or installation ID
and inspect the recorded update history for a selected installation.

<img src={"/docs/console/installation-history.jpg"} />

Snapshot-replacing databases such as S3 do not expose the Analytics capability
because concurrent event writes can conflict. The Console uses capability
detection—not a database name check—to omit Analytics navigation, routes, and
queries for every unsupported adapter.
