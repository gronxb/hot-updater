## Installation

Install Drizzle ORM and the SQLite driver as runtime dependencies. Install the
schema tooling and the driver's TypeScript declarations for development.

```package-install
npm install drizzle-orm better-sqlite3
npm install --save-dev drizzle-kit @types/better-sqlite3
```

For other databases, see [Drizzle's adapter documentation](https://orm.drizzle.team/docs/get-started).

Hot Updater treats Drizzle as an official database middle layer. The
`provider` option selects the concrete database dialect beneath Drizzle.

## Setup

Create a database connection file:

```typescript title="src/drizzle.ts"
import { drizzle } from "drizzle-orm/better-sqlite3";
import Database from "better-sqlite3";
import * as schema from "./hot-updater-schema";

const sqlite = new Database("./data/database.db");
export const db = drizzle(sqlite, { schema });
export { schema };
```

## Hot Updater Configuration

Create the Hot Updater instance:

```typescript title="src/hotUpdater.ts"
import { createHotUpdater } from "@hot-updater/server";
import { drizzleDatabase } from "@hot-updater/server/adapters/drizzle";
import { s3Storage } from "@hot-updater/aws";
import { db, schema } from "./drizzle";

export const hotUpdater = createHotUpdater({
  database: drizzleDatabase({ db, provider: "sqlite", schema }),
  storages: [
    s3Storage({
      region: "auto",
      endpoint: process.env.R2_ENDPOINT,
      credentials: {
        accessKeyId: process.env.R2_ACCESS_KEY_ID!,
        secretAccessKey: process.env.R2_SECRET_ACCESS_KEY!,
      },
      bucketName: process.env.R2_BUCKET_NAME!,
    }),
  ],
  basePath: "/hot-updater",
});
```

## Schema Generation

Generate the Drizzle schema for Hot Updater tables into `src`. The command can
bootstrap the missing `src/hot-updater-schema.ts` import on the first run.

```bash
npx hot-updater db generate src/hotUpdater.ts src --yes
```

Create the Drizzle Kit configuration that points to the generated schema:

```typescript title="drizzle.config.ts"
import { defineConfig } from "drizzle-kit";

export default defineConfig({
  dialect: "sqlite",
  schema: "./src/hot-updater-schema.ts",
  out: "./drizzle",
  dbCredentials: {
    url: "./data/database.db",
  },
});
```

Generate versioned migrations for review:

```bash
npx drizzle-kit generate
```

For local development, you can instead synchronize the generated schema
directly to the database:

```bash
npx drizzle-kit push
```

Run `hot-updater db generate` again after upgrading Hot Updater, then review a
new Drizzle migration before deploying it.

<Callout>
  Need to generate SQL without config? See [SQL
  Export](/docs/custom/database/sql-export) for standalone SQL generation with
  the `--sql` flag.
</Callout>

## CLI Configuration

Configure your CLI to use this self-hosted server.

```typescript title="hot-updater.config.ts"
import { defineConfig } from "hot-updater";
import { bare } from "@hot-updater/bare";
import { s3Storage } from "@hot-updater/aws";
import { standaloneRepository } from "@hot-updater/standalone";

export default defineConfig({
  build: bare({ enableHermes: true }),
  storage: s3Storage({
    region: "auto",
    endpoint: process.env.R2_ENDPOINT,
    credentials: {
      accessKeyId: process.env.R2_ACCESS_KEY_ID!,
      secretAccessKey: process.env.R2_SECRET_ACCESS_KEY!,
    },
    bucketName: process.env.R2_BUCKET_NAME!,
  }),
  database: standaloneRepository({
    baseUrl: "http://localhost:3000/hot-updater",
  }),
  updateStrategy: "appVersion",
});
```

<Callout type="warn">
  The `storage` plugin must match the `storages` in your server's
  `createHotUpdater`. Both use `s3Storage` in this example.
</Callout>

## API Endpoints

The server automatically creates these endpoints:

| Method | Endpoint                                                                              | Description                                                         |
| ------ | ------------------------------------------------------------------------------------- | ------------------------------------------------------------------- |
| GET    | `/hot-updater/version`                                                                | Get server version                                                  |
| GET    | `/hot-updater/fingerprint/:platform/:fingerprintHash/:channel/:minBundleId/:bundleId` | Check for updates by fingerprint                                    |
| GET    | `/hot-updater/app-version/:platform/:version/:channel/:minBundleId/:bundleId`         | Check for updates by app version                                    |
| GET    | `/hot-updater/api/bundles`                                                            | List bundles (query: channel, platform, limit, page, after, before) |
| GET    | `/hot-updater/api/bundles/:id`                                                        | Get bundle by ID                                                    |
| POST   | `/hot-updater/api/bundles`                                                            | Create bundles                                                      |
| PATCH  | `/hot-updater/api/bundles/:id`                                                        | Update a bundle                                                     |
| DELETE | `/hot-updater/api/bundles/:id`                                                        | Delete bundle                                                       |
| GET    | `/hot-updater/api/bundles/channels`                                                   | List all channels                                                   |
