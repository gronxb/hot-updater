## Prerequisites

- Cloudflare account at [cloudflare.com](https://cloudflare.com)
- Node.js 20+ installed locally
- An existing React Native app that already uses the Hot Updater CLI

## Installation

Install the Worker runtime dependencies.

```package-install
npm install @hot-updater/server @hot-updater/cloudflare hono
```

Install Wrangler and Cloudflare types.

```package-install
npm install -D wrangler @cloudflare/workers-types
```

If your TypeScript config does not already include Worker globals, add
`@cloudflare/workers-types` to `compilerOptions.types`.

```json title="tsconfig.json"
{
  "compilerOptions": {
    "types": ["@cloudflare/workers-types"]
  }
}
```

## Cloudflare Resources

Create a D1 database.

```bash
npx wrangler d1 create hot-updater-db
```

Create an R2 bucket for bundle storage.

```bash
npx wrangler r2 bucket create hot-updater-bundles
```

## Wrangler Configuration

Create `wrangler.toml` and bind D1 and R2 to the Worker.

```toml title="wrangler.toml"
name = "hot-updater-worker"
main = "src/index.ts"
compatibility_date = "2026-03-01"

[[d1_databases]]
binding = "DB"
database_name = "hot-updater-db"
database_id = "your-database-id-here"

[[r2_buckets]]
binding = "BUCKET"
bucket_name = "hot-updater-bundles"
```

Replace `database_id` with the value returned by `wrangler d1 create`.

## Worker Secret

Create a secret used to sign bundle download URLs.

```bash
npx wrangler secret put JWT_SECRET
```

## Worker Runtime Plugins

Use the Worker-specific subpath exports for the runtime:

- `@hot-updater/cloudflare` for the CLI provider in `hot-updater.config.ts`
- `@hot-updater/cloudflare/worker` for the Worker runtime in `src/hotUpdater.ts`

The Worker subpath exposes `d1Database` and `r2Storage` with the same names as
the CLI-side plugins, and also exports `verifyJwtSignedUrl` for signed bundle
downloads. These runtime exports read D1, R2, and `JWT_SECRET` from the request
context instead of calling the Cloudflare API directly.

## Hot Updater Configuration

Create the Hot Updater instance at module scope and keep the Worker env type
alongside it.

```typescript title="src/hotUpdater.ts"
import {
  d1Database,
  r2Storage,
  type RequestEnvContext,
} from "@hot-updater/cloudflare/worker";
import { createHotUpdater } from "@hot-updater/server/runtime";

export type CloudflareWorkerEnv = {
  DB: {
    prepare: D1Database["prepare"];
  };
  BUCKET: R2Bucket;
  JWT_SECRET: string;
};

export const HOT_UPDATER_BASE_PATH = "/api/check-update";

const resolveRequestOrigin = (context?: RequestEnvContext) => {
  const request = context?.request;

  if (!request) {
    throw new Error(
      "r2WorkerStorage requires a request to resolve publicBaseUrl.",
    );
  }

  return new URL(request.url).origin;
};

export const hotUpdater = createHotUpdater({
  database: d1Database(),
  storages: [
    r2Storage({
      publicBaseUrl: resolveRequestOrigin,
    }),
  ],
  basePath: HOT_UPDATER_BASE_PATH,
  routes: {
    updateCheck: true,
    bundles: false,
  },
});
```

## Worker Entry Point

Mount the update-check handler at the top level and pass `request` and `env`
into `hotUpdater.handler()`.

```typescript title="src/index.ts"
import { verifyJwtSignedUrl } from "@hot-updater/cloudflare/worker";
import { Hono } from "hono";
import {
  HOT_UPDATER_BASE_PATH,
  hotUpdater,
  type CloudflareWorkerEnv,
} from "./hotUpdater";

const app = new Hono<{ Bindings: CloudflareWorkerEnv }>();

app.mount(
  HOT_UPDATER_BASE_PATH,
  (request: Request, env: CloudflareWorkerEnv) => {
    return hotUpdater.handler(request, {
      request,
      env,
    });
  },
  {
    optionHandler: (c) => [c.env],
  },
);

app.get("*", async (c) => {
  const result = await verifyJwtSignedUrl({
    path: c.req.path,
    token: c.req.query("token"),
    jwtSecret: c.env.JWT_SECRET,
    handler: async (storageUri) => {
      const [, ...key] = storageUri.split("/");
      const object = await c.env.BUCKET.get(key.join("/"));

      if (!object) {
        return null;
      }

      return {
        body: object.body,
        contentType: object.httpMetadata?.contentType,
      };
    },
  });

  if (result.status !== 200) {
    return c.json({ error: result.error }, result.status);
  }

  return c.body(result.responseBody, 200, result.responseHeaders);
});

export default app;
```

## Security

<Callout type="warn">
With Hono `mount()`, keep `/api/check-update/app-version/*` and
`/api/check-update/fingerprint/*` public for React Native clients.

This example uses
`routes: { updateCheck: true, version: true, bundles: false }`, so
`/api/check-update/version` stays mounted while `/api/check-update/api/*` is
not mounted.

Leave `/api/check-update/version` public for diagnostics or set
`routes.version: false` if you want update-check-only routes. If you enable
bundle routes later, protect `/api/check-update/api/*` with Hono middleware,
but keep the update-check routes public.

The catch-all download route stays reachable, but each bundle download is still
guarded by the signed `token` verified with `JWT_SECRET`.
</Callout>

## Schema Generation

Generate the Hot Updater schema from the top-level configuration module.

```bash
npx hot-updater db generate src/hotUpdater.ts --yes
```

## Database Migration

Apply the generated migrations to D1.

```bash
npx wrangler d1 migrations apply hot-updater-db
```

## Development

Run the Worker locally.

```bash
npx wrangler dev
```

The update-check endpoint is available at
`http://localhost:8787/api/check-update`.

## Deploy

Deploy to Cloudflare Workers.

```bash
npx wrangler deploy
```

Your public update-check runtime will be available at
`https://hot-updater-worker.your-subdomain.workers.dev/api/check-update`.

## CLI Configuration

Configure the CLI to upload bundles to the same R2 bucket and write metadata to
the same D1 database.

```typescript title="hot-updater.config.ts"
import { bare } from "@hot-updater/bare";
import { d1Database, r2Storage } from "@hot-updater/cloudflare";
import { defineConfig } from "hot-updater";
import { config } from "dotenv";

config({ path: ".env.hotupdater" });

export default defineConfig({
  build: bare(),
  storage: r2Storage({
    bucketName: process.env.HOT_UPDATER_CLOUDFLARE_R2_BUCKET_NAME!,
    accountId: process.env.HOT_UPDATER_CLOUDFLARE_ACCOUNT_ID!,
    cloudflareApiToken: process.env.HOT_UPDATER_CLOUDFLARE_API_TOKEN!,
  }),
  database: d1Database({
    databaseId: process.env.HOT_UPDATER_CLOUDFLARE_D1_DATABASE_ID!,
    accountId: process.env.HOT_UPDATER_CLOUDFLARE_ACCOUNT_ID!,
    cloudflareApiToken: process.env.HOT_UPDATER_CLOUDFLARE_API_TOKEN!,
  }),
});
```

<Callout type="info">
In this Cloudflare Worker setup, bundle management happens through the CLI's
Cloudflare provider plugins. The Worker only serves update checks and signed
bundle downloads.
</Callout>
