## Usage

Use `HotUpdater.setCohort()` when you want to move a device into a specific
numeric or custom cohort.

```tsx
import { HotUpdater } from "@hot-updater/react-native";

function assignQaCohort() {
  HotUpdater.setCohort("qa");
  console.log(HotUpdater.getCohort()); // "qa"
}
```

## Behavior

- Persists the cohort on the device for future update checks.
- Accepts numeric cohorts in the range `1..1000`.
- Accepts lowercase slug-style custom cohorts such as `qa` and `vip-testers`.
- Normalizes the value before storing it.
- Throws an error when the value is invalid.

## Important Notes

- Call `HotUpdater.getCohort()` first if you need to restore the previous value
  later.
- Percentage rollout applies only to numeric cohorts.
- To keep a custom cohort such as `qa` receiving a bundle alongside numeric
  rollout, add that value to `Target Cohorts` in the console. This lets the
  same bundle stay available to `qa` even if you later raise numeric rollout
  from `0%` to `50%`.
