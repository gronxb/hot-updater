## Usage

`HotUpdater.reload()` applies the currently downloaded bundle immediately.

- If no new bundle is ready, it reloads the current app code
- If a bundle was downloaded, that bundle becomes active immediately
- The exact behavior depends on [`HotUpdater.setReloadMethod()`](./setReloadMethod)

```tsx
import { HotUpdater } from "@hot-updater/react-native";

async function syncUpdate() {
  const updateInfo = await HotUpdater.checkForUpdate({
    updateStrategy: "appVersion",
  });

  if (!updateInfo) return;

  await updateInfo.updateBundle();

  if (updateInfo.shouldForceUpdate) {
    await HotUpdater.reload();  // [!code hl]
  }
}
```

## Reload method

By default, `HotUpdater.reload()` uses the normal React Native reload path.

If Android release builds crash during reload because native modules touch the React instance while it is being recreated, prefer [`HotUpdater.setReloadMethod("processRestart")`](./setReloadMethod).
